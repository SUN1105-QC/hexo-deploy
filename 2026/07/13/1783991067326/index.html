1. 部署总览

1.1 手册目标

本手册用于指导从零开始完成一个典型 Web 项目的部署上线工作，目标包括：

  • 购买并准备部署所需账号与资源
  • 配置服务器基础环境
  • 部署数据库、后端服务、前端资源
  • 配置 Nginx、域名与 HTTPS
  • 建立基础运维能力，如日志、备份、监控与自动恢复

1.2 总览目录

  1. 准备账号与基础资源
  2. 购买域名并完成基础解析准备
  3. 创建并初始化 Lightsail 服务器
  4. 部署 MySQL 8 数据库
  5. 部署后端服务
  6. 部署前端并配置 Nginx
  7. 配置域名与 HTTPS
  8. 配置日志、备份、监控与运维
  9. 验证上线结果与后续维护

1.3 整体流程图

2. 准备账号与基础资源

2.1 本章目标

完成部署前所需平台账号和基础资源准备，为后续购买域名、创建服务器和代码部署打下基础。

2.2 需要准备的账号

通常需要以下账号：

  • AWS 账号
  • 域名服务商账号
  • GitHub / GitLab 代码仓库账号
  • 常用邮箱
  • 可用于支付的银行卡或信用卡
  • SSH 工具（如 Windows Terminal、PuTTY、MobaXterm、iTerm2）

2.3 AWS 账号申请

平台地址

  • https://aws.amazon.com/

操作步骤

  1. 打开 AWS 官网
  2. 点击 Create an AWS Account
  3. 输入邮箱、账号名称、密码
  4. 完成邮箱验证
  5. 填写联系人信息
  6. 填写支付方式
  7. 完成手机验证
  8. 选择基础支持计划
  9. 登录 AWS 控制台

建议项

  • 使用公司邮箱注册
  • 开启 MFA 多因素认证
  • 根账号仅用于重要管理操作
  • 日常运维建议创建 IAM 子账号

初始设置建议

  • 配置 IAM 用户
  • 设置账单提醒
  • 开启 MFA
  • 记录账号信息并安全保管

2.4 域名服务商账号申请

可选平台

  • Namecheap
  • GoDaddy
  • 阿里云万网
  • 腾讯云域名
  • Cloudflare Registrar

操作步骤

  1. 打开域名服务商官网
  2. 点击注册
  3. 使用邮箱完成注册
  4. 完成邮箱验证或手机验证
  5. 进入用户后台
  6. 完成实名认证（如平台要求）

2.5 代码仓库账号准备

可选平台

  • GitHub
  • GitLab
  • Gitee

操作建议

  • 确保项目代码已推送到仓库
  • 准备好部署分支
  • 如为私有仓库，准备 SSH Key 或 Access Token
  • 为服务器部署用户配置仓库访问权限

2.6 建议的初始安全设置

  • 所有重要平台开启 MFA
  • 不共用主账号密码
  • 将支付相关账号与技术操作账号分离
  • 将服务器登录密钥妥善保管
  • 不将敏感信息直接写入公开代码仓库

2.7 本章验证方式

  • 能成功登录 AWS 控制台
  • 能成功登录域名管理后台
  • 能成功登录代码仓库平台
  • 可以访问项目仓库
  • 已准备好支付和邮箱验证能力

2.8 生活化理解

这一章像是：

先把开店需要的各种证件、平台后台账号和管理权限办齐。

你还没有开始真正装修店面，但至少要先有：

  • 商场管理后台账号
  • 招牌购买渠道
  • 仓库资料管理入口
